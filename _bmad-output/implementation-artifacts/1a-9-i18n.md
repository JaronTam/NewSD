**§3.7 E27 missing key fallback**: `t(key, lang)` 实现:

- key 在 dict 存在 + `dict[key][lang]` 有值 -> 返回 `dict[key][lang]`.
- key 在 dict 存在 + `dict[key][lang]` 缺值(缺翻译) -> fallback `dict[key].en`(English 兜底) + `console.warn(`[i18n] missing translation: ${key} for ${lang}`)`.
- key 完全不在 dict(dict bug) -> `console.warn(`[i18n] missing key: ${key}`)` + 返回 key 名(最后防线; AC-12 grep 校验 dev 期防此情况, 运行时兜底不崩).
- **E27 三不**: 非空串(fallback English 或 key 名, 非空) / 非 key 名裸露(正常运行靠 dict 完整 + grep 守卫, 不显示 key 名给用户; 缺翻译 fallback English 非裸 key 名) / 非崩(始终返回 string, 不 throw).

# Lovable Prototype Prompt — ASCII SD 模拟器（工作流 B 原型）

> 用途：在 Lovable 中生成一个**可点击、能玩**的 prototype，用来锁定「ASCII 画布 + 游戏化」的核心交互质感。
> 原型是「参考答案」不是「生产代码」——数值内核/协作后端一律 mock，只求交互手感对。
> 粘进 Lovable 时，整段复制即可。

---

## 你的角色

你是一个资深前端工程师，擅长 React + TypeScript + Tailwind + Canvas 2D。
你要构建一个**赛博朋克风格的 ASCII 系统动力学模拟器** prototype。
重点是**交互质感和视觉美学**，不是工程完整性。能点、能动、有爽快反馈即可。

## 产品是什么（30 秒理解）

一个无限画布工具，用户用 ASCII 字符图元搭「存量-流量模型（SFD）」，点播放看数值随时间变化，过程收集游戏化徽章。
类比：Vensim/Stella 的功能骨架 + 赛博朋克终端的皮 + 轻度游戏化的魂。

## 技术栈（硬性）

- React + TypeScript + Tailwind
- 画布渲染用 **Canvas 2D**（不要用 DOM 网格，不要用 SVG，不要引第三方图表库）
- 等宽字体（`Courier New` / `JetBrains Mono`），关闭抗锯齿保像素风
- 状态用 Zustand（或 React state，prototype 不纠结）
- 全部本地、单机、mock 数据，不接后端、不接 Supabase

## 视觉美学（这是核心卖点，务必到位）

- **赛博朋克暗色主题**：背景近黑（`#0a0e14` 级），网格线极暗（`#1a1f2e`）
- **霓虹高亮字符**：
  - 存量方框边框 `#00ffd5`（青）
  - 流量箭头 `#ff2e88`（品红）
  - 源/汇云朵 `#7c3aed`（紫）
  - sparkline `#39ff14`（霓虹绿）
  - 选中态 `#ffd700`（金）+ 闪烁
- 字符带轻微 `text-shadow` 霓虹辉光（Canvas 上用 `shadowBlur`，但克制，别全局开）
- 终端风 UI：边框用 `┌┐└┘─│` 字符画，按钮像终端按键
- 整体气质：像在玩一个 90 年代 BBS 里的高阶建模玩具

## 必须实现的 FR（从 PRD 抽取，按优先级）

### P0 — 画布与图元（无这层就不是这个产品）

**FR-CANVAS-1 无限画布导航**
- 中键拖拽 或 空格+左键 平移
- 滚轮缩放，minZoom=0.05，maxZoom=20
- 世界坐标 Float64，屏幕投影用 3×2 仿射矩阵（prototype 可简化，但要有 world↔screen 转换）

**FR-CANVAS-3 ASCII 字符渲染**（最关键）
- 三类图元，全部字符画：
  - **存量 Stock**：方框字符 `┌┐└┘─│` 包围，内显名称+当前值+单位，如：
    ```
    ┌────────┐
    │ 水位   │
    │ 1234 升│
    └────────┘
    ```
  - **源/汇 Cloud**：ASCII 云朵：
    ```
      .--.
     (    )
      `--'
    ```
  - **流量 Flow**：`→` 箭头连线。可变流量中点画 `▼`，常数流量相交处画 `○`
- 所有文本严格对齐字符网格（等宽，整数像素定位）

**FR-ELEM-1/2/3/4 图元系统**
- 存量属性：name, initialValue, units（如"人""升""元"）, allowNegative, currentValue(运行时)
- 源/汇：边界元素，无限容量
- 流量：fromId→toId，formula 字符串（如 `水位*0.1`、`5`），isVariable 布尔
- 每个元素周边有连接端口，创建流量时自动吸附

**交互**：拖拽移动、点击选中、双击编辑、调整大小（存量）

### P0 — 仿真与可视化（壳 + mock 内核）

**FR-SIM-1（mock 版）求解器**
- **不要实现 BDF/牛顿迭代**。prototype 用最朴素显式欧拉：`stock[t+dt] += sum(flows) * dt`
- 顶部工具栏控制：▶ 播放 / ⏸ 暂停 / ⏹ 重置 / ⏭ 单步
- dt 选择器：[0.01, 0.1, 0.5, 1.0]
- 公式求值：**禁用 eval**，写个极简安全解析器（只支持 `+ - * / ()` 数字和存量名引用即可）。prototype 够用就行。

**FR-SIM-VIZ-1 内联 ASCII sparkline**（第二关键）
- 每个存量方框旁实时显示 ASCII 趋势，字符集 `▁▂▃▄▅▆▇█`
- 数值自动归一化到字符高度档位
- 保留最近 N 步（默认 32），与 dt 解耦
- **存量方框同时显示当前数值+单位**（不归一化），如 `[水位] 1234 升` 旁边一条小绿条
- 纯字符渲染，不引图表库

### P0 — 游戏化（这是要试出「魂」的部分）

**FR-GAME-1 总开关**
- 设置里一个开关，关掉后徽章完全不显示不触发

**FR-GAME-2 行为徽章系统**
- 基于建模动作触发（不依赖仿真结果质量），内置至少这些触发器：
  - 「初探」首个存量
  - 「连线」首次连接流量
  - 「点火」首次跑仿真
  - 「织网」连接 10 个图元
  - 「成模」完成首模型（≥1 存量 + ≥1 流量 + 成功跑一次仿真）
- 徽章用 **ASCII 风格图标** 呈现（自己用字符画几个，如 `★ ◆ ⬢ ⚡` 配霓虹色）
- 触发时**即时反馈**：右上角弹一个 toast，徽章图标 + 名称 + 一行描述，带霓虹闪烁动效，2-3 秒自动消失
- 徽章记录侧栏可查看已解锁/未解锁

### P1 — UI chrome（让原型看起来像成品）

**FR-UI-1 顶部工具栏**：文件(新建/打开/保存 可mock空操作) / 编辑(撤销/重做/复制/粘贴/删除) / 工具切换(选择/存量/源汇/流量) / 模拟控制 / dt 选择器 / 缩放指示器

**FR-UI-2 右侧属性面板**：选中元素显示属性。存量编辑 name/initialValue/units/allowNegative；流量编辑 formula + isVariable 切换 + 显示派生单位(只读)。公式语法错误红色高亮。

**FR-UI-3 底部状态栏**：模拟时间计数器 / 图元计数 / FPS 指示器 / 量纲校验概要（mock，可写死 `量纲: 3/5 一致 · 2 软警告`）

**FR-UI-4 中/英双语切换**：界面 chrome 文本中英可切，即时生效。用户输入的 name/公式不切换。默认跟随 `navigator.language`。

## 明确不要做的（省 Lovable 算力，避免跑偏）

- ❌ 不要实现隐式求解器/BDF/牛顿迭代/自动微分（mock 成显式欧拉）
- ❌ 不要实现 CRDT/Yjs/多人协作（单机即可，留个「在线用户数:1」的假状态）
- ❌ 不要实现 R 树空间索引（prototype 几十个图元，线性遍历够用）
- ❌ 不要实现量纲校验推导（状态栏写死概要即可，公式编辑器语法错误检查做个极简版）
- ❌ 不要做房主迁移、跨画板粘贴协议、SQLite 持久化
- ❌ 不要引 ECharts/D3/任何图表库
- ❌ 不要做移动端响应式
- ❌ 不要做挑战关卡（FR-GAME-3 是后续项）

## 初始场景（让原型打开就有东西可玩）

启动时预置一个**人口模型** demo：
- 一个源（出生）
- 一个存量「人口」initialValue=1000 units=人
- 一个汇（死亡）
- 流量：出生 = 人口*0.05（可变，▼），死亡 = 人口*0.02（可变，▼）
- 默认 dt=0.1，时间单位=年

用户打开就能点播放，看人口 sparkline 指数上升，顺手解锁「初探」「连线」「点火」「成模」四个徽章——一气呵成 demo 完整故事。

## 验收（prototype 算成功的标准）

1. 打开看到赛博朋克暗色画布 + 预置人口模型，霓虹字符辉光到位
2. 滚轮缩放、中键平移流畅，字符不糊
3. 能从工具栏选「存量」往画布上放新图元，能连流量
4. 点 ▶ 播放，存量方框的数值实时跳动，旁边 sparkline 实时生长
5. 做前几个动作时，右上角依次弹出徽章 toast，有霓虹动效
6. 设置里关掉游戏化开关，徽章 toast 不再出现
7. 中/英切换即时生效
8. 整体气质像「赛博朋克终端里的高阶建模玩具」，不像普通 CRUD 后台

# 系统动力学 ASCII 模拟器产品需求文档

---

## 一、产品概述

### 1.1 产品愿景

一款赛博朋克风格、基于 ASCII 字符渲染的多人协作系统动力学建模平台。用户在无限画布上使用字符图元构建存量流量模型（SFD），运行数值模拟，并实时多人协作。因果回路图（CLD）作为后续能力规划，MVP 不含。

产品差异化特点：
- **ASCII 美学**：赛博朋克暗色主题 + 霓虹高亮字符，纯字符渲染而非矢量图
- **结构即内容**：文本表示既是模型数据也是可视化呈现
- **沉浸式交互质感**：类游戏工具的爽快响应、像素级精确反馈、状态机驱动的明确操作
- **可选游戏化层**：行为徽章激励建模探索（挑战关卡为后续规划），可由用户关闭

> **硬约束**：本产品不因内部数值深度妥协交互简洁性；交互设计始终以"架构即内容"为不可协商的门槛 KPI。
>
> 双模压力测试示例（非矛盾，压力点已转形为单一透明手势）：刚性化学动力学需 BDF 隐式求解 → 复杂度封装 Wasm 内核，交互层仅"播放"；若步长崩溃或残差超阈 → 自动降级后向欧拉或显式试探步，状态层仅暴露"求解器自适应调节中"指示器（sparkline 旁闪烁黄点），用户无感；若自动降级仍失败（罕见）→ 弹出单按钮"简化模型以继续"，点击后系统自动执行预设降级组合（减少存量、降低耦合），用户不经手求解器档位。用户全程仅"播放—暂停—一键恢复"范式，零新增控件——硬约束由此自证。

### 1.2 问题陈述

现有系统动力学建模工具（Vensim、Stella、Insight Maker）存在以下痛点：
- 界面过于复杂，入门门槛高
- 缺乏实时多人协作功能
- 设计风格千篇一律，建模过程缺乏沉浸感
- 不支持轻量游戏化学习体验
- 仿真工具普遍使用陈旧架构与渲染方式，缺乏现代工程实践

#### 现状工具对比矩阵

| 工具 | 交互复杂度暴露层 | 数值内核 | 不收敛处理 |
|------|----------------|---------|-----------|
| Vensim | 多级菜单 + 概念前置暴露 | 显式 RK4 / DDE | 手动调参 / 换算法 |
| Stella | 图层 + 仿真参数面板前置 | 显式欧拉 / RK4 | 手动调求解器设置 |
| Insight Maker | Web 表单 + 配置项繁多 | 显式求解 | 手动调时间步长 |
| **本工具** | **仅三步：拖图元 → 写公式 → 看趋势** | **隐式 BDF + 牛顿 + 自动微分（透明）** | **自动降级 + 一键简化，用户无感知 BDF/显式切换** |

> **双模架构承诺**
>
> **定位一致性声明**：本工具不对用户实施"教学"，而是提供高保真模拟沙盒，使复杂系统运行逻辑在游戏化交互中被自行感知。
>
> 本产品作出双重承诺，二者并列、不可互替：
>
> - **交互模**——低门槛探索体验：ASCII 图元拖拽即建模，可选游戏化，默认简化界面。
> - **数值模**——专业级隐式求解内核：后向欧拉 / BDF + 牛顿迭代 + 自动微分，保障数值洞察可靠性。
>
> 两模经透明分层共存：数值内核为交互模的透明底层，L1 用户不感知求解器；L3 用户可解锁刚性系统求解。交互简洁性与数值深度的张力，由 §1.1 硬约束为门槛、§5.1 逃生阀为回调，作架构级处置。

### 1.3 目标用户

MVP 主打游戏化探索场景（入门层降低认知负荷、与可选游戏化层——行为徽章、后续挑战关卡——天然契合）：

- **游戏者**：在无限画布上构建存量流量模型、运行模拟、探索系统行为；游戏化徽章激励探索

> 产品受众合并为单一"游戏者"，按成熟度分层（L1 入门 / L2 进阶 / L3 专业）承载不同探索深度，不再按角色单列。

游戏化层对所有用户可选开关（FR-GAME-1）。

#### 游戏者成熟度分层与数值内核可见性

承接 §1.2 双模架构承诺，同一"游戏者"按成熟度分层暴露数值内核——不同成熟度看到不同可见度，而非不同产品：

| 层级 | 成熟度 | 交互面 | 数值内核可见性 |
|------|--------|--------|---------------|
| **L1 入门** | 零基础探索者 | 默认简化界面：拖图元、写公式、看 sparkline、收徽章 | 完全透明（求解器在后台；量纲不一致仅软警告可忽略） |
| **L2 进阶** | 系统分析者 | 默认界面 + 量纲校验概要、熔断/降级提示、单位显示 | 半透明（可查看量纲结果、感知 FR-SIM-8 自适应降级黄点） |
| **L3 专业** | 高级建模者 | 解锁刚性系统场景、自定义时间单位 | 可见（感知隐式求解器档位、可触达 §5.1 逃生阀降级） |

> 分层为产品定位说明，非功能开关。MVP 不实现 L1/L2/L3 切换 UI；默认即 L1 体验，L2/L3 能力按既有 FR 渐进暴露，不新增切换控件——与 §1.1 硬约束一致。

#### 默认可见性规约（L1 兑现锚点）

分层承诺的可验收落地——规定**默认配置**下各层元素的可见性，使"L1 用户不感知求解器"成为可判定的 spec 而非模糊承诺。违反即 PRD 缺陷。

| 可见性档 | 元素 | 默认状态 | 触达方式 |
|---------|------|---------|---------|
| **L1 隐藏** | 求解器档位选择器（BDF/后向欧拉/显式） | 不渲染 | FR-SIM-1/FR-SIM-8 内部自适应，无用户入口 |
| **L1 隐藏** | 量纲校验硬阻断开关 | 不渲染 | FR-SIM-7 定为软警告，无硬阻断开关 |
| **L1 隐藏** | 雅可比/牛顿迭代参数（容差、maxIterations） | 不渲染 | 求解器内部常量 |
| **L1 隐藏** | 熔断阈值手动调整入口 | 不渲染 | FR-SIM-6 默认值生效，入口归 L2 |
| **L2 渐显** | 量纲校验概要、单位显示 | 触发后可见 | 进阶操作（如打开量纲面板）唤起，见 FR-UI-3/FR-ELEM-1 |
| **L2 渐显** | 熔断/降级提示 | 事件触发可见 | FR-SIM-6 HALTED / FR-SIM-8 黄点，无需用户开启 |
| **L3 解锁** | 刚性系统场景、自定义时间单位 | 显式开启 | FR-UI-5 等 |
| **L3 解锁** | §5.1 逃生阀降级配置 | 架构阶段触发 | 非运行时 UI |

**验收**：默认配置首次启动，上表"L1 隐藏"四项均不在 UI 渲染；"L2 渐显"项仅在对应事件/进阶操作触发时出现，无永久开关；"L3 解锁"项需用户显式开启方可见。此项与 FR-SIM-8（求解器透明降级）共同构成 §1.1 硬约束的可验证证据链。

#### 用户旅程（脚注式举例）

- **L1 入门** [例:人口模型→仅写一个方程：出生率=0.02×人口，点播放看 sparkline 上升，全程不接触"隐式求解器"四字]
- **L1 入门·量纲错误** [例:感染率写成 0.3×易感者→状态栏"1 软警告"可忽略继续，无须理解量纲即可建模]
- **L3 专业·刚性系统** [例:刚性化学动力学→系统自动尝试 BDF，若极端刚性致失败，弹出"简化模型"按钮，点击后立即继续，游戏者仍只经历"播放-暂停-继续"]
- **L2 进阶·量纲分析** [例:SIR 模型→开启量纲概要查看"感染率预期 1/时间"，量纲识读可见]

### 1.4 成功指标

| 指标 | 目标值 |
|------|--------|
| 1000 个图元时画布帧率 | ≥ 60 FPS |
| 协作延迟 P95 | ≤ 100 毫秒 |
| 模拟运算速度 | 100 个存量 ≥ 100 步/秒 |
| 单画布最大并发用户 | 10 人 |
| 首次建模时间（新手上路） | ≤ 5 分钟 |
| 单会话模型迭代次数（探索深度） | 中位数 ≥ 3 次（同一会话内对模型修改并重跑，衡量"复杂系统直觉唤醒"是否被激发） |
| 徽章解锁率（游戏化激励成效） | 行为徽章首次会话解锁 ≥ 1 枚的用户占比 ≥ 60% |
| 模型保存/加载失败率 | < 0.1% |
| 多客户端间模拟数值偏差 | < 0.001% |

**反指标（必须避免）**：
- 剪贴板畸形/非法图元结构注入 = 0（任意构造的剪贴板 payload 不得注入非法图元结构；粘贴通道不经 eval，不承诺密码学认证）
- 房主迁移导致的仿真状态清零 = 0（迁移须从会话内快照续跑，不得回退 t=0）

## 二、设计理念核心（来自 Idea 文件的关键决策）

### 2.1 技术选型核心逻辑

**工程本质判断**：系统动力学画板 = 无限视口投影矩阵 × R 树空间索引 × CRDT 乐观并发

**因果链推导路径**（来自 Idea 文件香农零熵协议格式）：

1. **渲染层选型**：
   - 事实：DOM 节点超过 80×60 网格高频刷新引发 Layout 抖动
   - 因果映射 → 必须用 VRAM 双缓冲区（文本缓冲 + 颜色缓冲）
   - 边界约束 → 霓虹字体 shadowBlur 极端消耗 GPU
   - 结论：Canvas 2D 定点渲染二维字符数组，关闭抗锯齿保证像素风

2. **模拟引擎选型**：
   - 事实：用户在面板配置复杂流量公式
   - 因果映射 → 前端必须动态解析代数环
   - 边界约束 → 防止 JavaScript 注入 + 提升高频迭代性能
   - 结论：Rust mexpr 库构建 AST，Wasm 层拓扑排序检测代数环并断流；求解器、量纲校验、自动微分等数值子系统选型见 addendum §7

3. **后端选型**：
   - 事实：ASCII 结构天然高压缩率
   - 因果映射 → 画板数据本质是纯文本，无需复杂解析
   - 边界约束 → 轻量化、极速读写，且 MVP 为单节点、10 人匿名协作
   - 结论：服务端 SQLite 存储画板与 CRDT 快照（单文件、服务进程内访问），Presence 与仿真态走服务进程内存。PostgreSQL、Redis 等水平扩展组件留作后续路线图（详见 addendum）

### 2.2 七大数学约束（来自 Idea 文件深度分析）

| 约束名称 | 技术原理 | 解决的问题 |
|----------|----------|------------|
| **存量断路器算法** | 拓扑排序前强行剪断所有存量流出边 | 区分"积分反馈环"与"代数死锁环" |
| **单次求值原则** | 所有流量速率只算一次，查表供所有存量使用 | 消除浮点误差累计造成的物质凭空产生/消失 |
| **非负钳制机制** | 存量<0时限制流出，级联影响下游 | 水箱空了就不能再流了的物理现实 |
| **DELAY 隐式存量** | 延迟函数不做历史队列，转为串联幽灵存量 | 保证隐式求解器联立求解时的连续性 |
| **双时间轴分离** | UI 时间轴 ≠ 仿真时间轴 | 仿真数值异常不传导到画布渲染 |
| **NaN 熔断保护** | Wasm 每步收敛后做 NaN/Inf 位运算检查 + 相对量级突变检查（FR-SIM-6） | 正反馈系统数值上溢时的紧急停机 |
| **两遍扫描初始化** | Pass0 算初始值 → Pass1 焊死后推导流速 | t=0 时刻时序死锁 |

## 三、功能需求

### 3.1 画布渲染系统

**FR-CANVAS-1**：无限画布导航
- 平移：中键拖拽 或 空格 + 左键
- 缩放：滚轮、双指捏合、或缩放控件
- 缩放约束：minZoom = 0.05 (5%)，maxZoom = 20 (2000%)
- 世界坐标系：Float64 精度
- 屏幕坐标投影：3×2 仿射变换矩阵

**FR-CANVAS-2**：网格吸附对齐
- 基础网格单位：1 世界单位 = 1 字符单元格
- 屏幕空间吸附容差：恒定 8px（与缩放无关）
- 吸附容差公式：`snapTolerance = 8 / currentZoom`（单位：世界单位；与网格世界坐标比较，换算回屏幕恒为 8px）
- 用户可配置网格步长

**FR-CANVAS-3**：ASCII 字符渲染
- 所有元素使用等宽字体渲染（Courier New 或等效字体）
- 3 类图元元素（与 3.2 FR-ELEM 一一对应）：
  - **源/汇 (Cloud)**：ASCII 云朵字符画 `  .--.  ` / ` (    ) ` / `  `--'  `
  - **存量 (Stock)**：方框绘制字符 `┌`, `┐`, `└`, `┘`, `─`, `│`
  - **流量箭头 (Flow)**：箭头 `→`，依据 `isVariable` 渲染两种变体
    - 可变流量（`isVariable=true`）：箭头中点处绘制倒三角 `▼`
    - 常数流量（`isVariable=false`）：箭头线与圆圈 `○` 相交
- ▼/○ 为 flow 元素的渲染变体，非独立图元类型
- 反馈回路不作为独立图元标记——它是 SFD 模型的涌现结果，由存量与流量连接自然形成
- 文本标签严格对齐字符网格

**FR-CANVAS-4**：空间索引与视口剔除
- 所有画布元素使用 R 树空间索引
- 每帧仅查询并绘制与当前视口相交的元素
- 脏矩形追踪：仅重绘变化区域
- 目标性能：10,000 个元素维持 30 FPS 以上

### 3.2 模型图元系统

**FR-ELEM-1**：存量元素
- 属性：
  - `id`：UUIDv4
  - `type`："stock"
  - `x, y`：世界坐标（左上角）
  - `width, height`：字符网格尺寸
  - `name`：显示标签文本
  - `initialValue`：初始数值
  - `units`：单位字符串（如 "人"、"元"、"升"），用于量纲校验与显示
  - `currentValue`：运行时模拟值（不持久化）
  - `allowNegative`：布尔值（默认 false）
- 渲染：ASCII 方框居中显示名称、`currentValue` 与 `units`（如 `[水位]\n 1234 升`）；仿真未运行时显示 `initialValue`；无 `units` 时仅显示数值（与 FR-SIM-VIZ-1 一致）
- 交互：拖拽移动、调整大小、点击选中、双击编辑

**FR-ELEM-2**：源/汇元素
- 属性：
  - `id`：UUIDv4
  - `type`："source" 或 "sink"
  - `x, y`：世界坐标
  - `name`：可选标签
- 渲染：ASCII 云朵图案
- 行为：边界元素，无限容量（源提供无限流入，汇吸收无限流出）

**FR-ELEM-3**：流量连接器
- 属性：
  - `id`：UUIDv4
  - `type`："flow"
  - `fromId`：起始元素 ID
  - `toId`：目标元素 ID
  - `formula`：速率表达式（如 "人口 * 0.1"、"5"）；公式中的常数可带单位标注，语法 `数值 [单位]`（如 `0.1 [1/year]`、`5 [人/year]`）
  - `units`：流量单位，自动派生为目标存量 `units / 时间单位`（如存量"人"+ 时间单位"year" → 流量"人/year"）；用户不可直接编辑，由量纲校验推导
  - `isVariable`：布尔值（true 渲染三角，false 渲染圆圈）
- 流量方向由 `fromId → toId` 表达；流入/流出语义在公式求值时自然体现，不设独立极性字段（CLD 的 +/- 极性概念不适用于 SFD）
- **公式对存量的引用**：公式内部以 stockId 引用存量（如 `@<uuid> * 0.1`），编辑器显示为对应 name；重命名存量只改 name 不动 id，公式引用零断链；协作并发重命名安全
- **重名软警告**：允许同名存量（不强制唯一），但编辑器检测到重名时给软警告提示可读性风险；公式量纲推导（FR-SIM-7）以 id 精确解析被引用存量的 units
- 渲染：
  - Bresenham 直线算法在字符网格间寻路
  - 目标端绘制箭头 `→`
  - 内联显示公式标签
  - 可变流量在中点绘制 `▼` 标记
  - 常数流量在相交处绘制 `○` 标记

**FR-ELEM-4**：连接端口
- 每个元素在其周边有预定义的连接点
- 创建连线时流量自动吸附到端口
- 拖拽元素时端口位置随之更新

### 3.3 模拟引擎系统

**FR-SIM-1**：数值求解器
- 求解器类型：隐式求解器（后向欧拉 / BDF），每步经牛顿迭代收敛，适用于刚性系统
- 时间单位：模型级设置（年/月/日/时/分/秒 等，见 FR-UI-5 模型设置）；dt 以该时间单位计量
- 时间步长：可配置（默认 dt = 0.1 个模型时间单位）
- 雅可比来源：由公式 AST 自动微分得到（与 FR-SIM-7 量纲 AST 复用）；每步牛顿迭代含 LU 分解线性求解与收敛判定
- 手动控制：播放/暂停/单步/重置 按钮
- 房主计算架构：仅房间房主运行模拟运算
- 注：隐式求解器为 MVP 含项，工程复杂度高（自动微分 + 牛顿迭代 + 线性求解），架构阶段需专项规约

**FR-SIM-2**：存量断路器（代数环检测）
- 编译时检测并解析代数环：
  1. 构建所有依赖关系的有向图
  2. 暂时移除所有存量节点的流出边
  3. 若剩余图仍有环 → 检测到代数环，高亮并拒绝运行
  4. 若无环 → 依赖图合法，交由隐式求解器联立求解所有存量（拓扑序仅用于环检测与求值依赖合法性确认，不作为显式逐个求值的顺序）
- 用户反馈：冲突链路红色高亮 + 弹窗解释
- 隐式求解器（FR-SIM-1）下仍保持严格剪边：检测到代数环一律拒绝运行，不因隐式法可解而放行——代数环在 SD 中通常为建模错误，严格剪边有助于在探索中直观区分"合法积分反馈"与"代数死锁"

**FR-SIM-3**：流量守恒原则
- 单收敛值原则：每个时间步收敛后，每个流量产出单一一致的速率值，所有连接该流量的存量引用同一个收敛速率值
- 隐式求解器（FR-SIM-1）下，流量在牛顿迭代内部为求残差/雅可比多次求值以收敛，此迭代过程属求解器实现细节，不构成对守恒原则的违反；约束作用于收敛结果而非迭代过程
- 防止浮点漂移和物质守恒违规（不同存量引用不同速率导致物质凭空产生/消失）

**FR-SIM-4**：非负钳制机制
- 对于 `allowNegative = false` 的存量：
  - 在隐式求解器（FR-SIM-1）的牛顿迭代中施加 `stock ≥ 0` 约束：当迭代解会使存量转负时，将该存量的流出约束为不越过零（投影到可行域）
  - 钳制后的实际流出速率不超过使存量恰好归零的速率
  - 级联效应：被钳制流量的下游流量相应重算；在雅可比中体现为活动约束的方程结构变化
  - 绝不发生"幽灵渗漏"
- 实现规约：非负钳制与隐式牛顿迭代叠加构成带约束非线性求解（投影法或活动集法），为架构阶段须专项规约的难点（见 addendum §2）

**FR-SIM-5**：DELAY 函数实现
- DELAY 内置函数不作为历史队列实现
- 而是作为串联的隐式存量构建
- 例如：`DELAY3(X, 6.0)` 在 AST 编译阶段插入 3 个串联的一阶微分方程
- 确保隐式求解器（FR-SIM-1）的连续性——DELAY 展开为存量后纳入统一联立求解

**FR-SIM-6**：数值溢出熔断（刚性系统经 FR-SIM-1 隐式求解器正确求解，本条为兜底保护）
- 满足以下任一条件时模拟自动暂停（相对量级阈值，适配 MVP 无单位场景）：
  - 任一存量绝对值超过 `max(所有存量初始绝对值) × 1e6`（相对初始量级上溢 6 个数量级）
  - 任一存量值为 `NaN` 或 `Infinity`
  - 相邻步相对变化率超过 `1e3`（`|value[t+dt] − value[t]| / max(|value[t]|, ε) > 1000`，ε 为防除零小常数）
- 阈值为 MVP 默认值，可由用户在设置中调整；调参引导（避免盲目调值损伤模型洞察可靠性）：
  - `1e6`（相对初始量级上溢 6 个数量级）：覆盖多数常规模型（人口/库存/流量平衡类，量级变化通常 ≤3–4 个数量级）。**上调场景**：强正反馈/指数增长模型（如细菌培养、复利），合法解本身可跨越 6 个数量级，默认值会误熔断——建议按预期峰值量级上调至 `1e8`–`1e10`。**下调场景**：一般不需要；若模型本应稳态却频繁接近 `1e6`，通常说明模型本身有发散 bug，应排查而非下调阈值。
  - `1e3`（相邻步相对变化率）：防止单步数值跳变。刚性系统初始瞬态或指数衰减近零阶段（`|value[t]|` 极小，ε 主导分母）可能合法触发——若确认是刚性瞬态而非数值病态，可临时上调至 `1e4`–`1e5`，瞬态结束后恢复默认。
  - ε（防除零小常数）：默认 `1e-12`；仅当模型合法涉及极小绝对值（如浓度趋零的化学反应）时才需下调，否则保持默认以防除零保护吞掉真实病态信号。
- 用户通知：ASCII 弹窗显示 `[SYSTEM HALTED: NUMERICAL OVERFLOW AT t=<实际时间>s]`

**FR-SIM-7**：量纲一致性校验（MVP）
- 编译期对每个流量公式做量纲推导：遍历该流量的结构化 AST（与 FR-COLLAB-1 CRDT 存储、FR-SIM-1 自动微分共享同一棵 AST，引用节点承载 stockId 供量纲精确解析），逐子表达式推导单位，最终推导出公式整体的单位
- 比对：推导出的公式单位须等于该流量的目标单位（目标存量 `units / 时间单位`）
  - 一致：通过
  - 不一致：软警告——公式旁红色高亮 + 提示预期单位与实际推导单位（如"预期 人/year，推导得 人"），**不阻断运行**（允许无量纲或部分校验场景继续建模），但用户须知晓结果可能无物理意义
- 常数单位标注参与推导：`0.1 [1/year]` 使 `人口(人) × 0.1(1/year)` 推导为 `人/year`，与流量目标单位一致
- 无 units 的存量视为无量纲；无量纲存量参与的流量，其目标单位为 `1/时间单位`
- 校验在公式编辑时实时进行（FR-UI-2 实时校验扩展）

**FR-SIM-8**：求解器自适应降级 + 一键简化（MVP）
- **背景**：兑现 §1.1 硬约束——数值深度不增加交互复杂度。不收敛作为双模架构压力测试点，转形为单一透明手势，而非要求用户理解求解器档位。
- **默认策略**：FR-SIM-1 BDF 步进内嵌收敛监控（残差范数 + 步长收缩率）；步长崩溃或残差超阈时，**自动**降级为后向欧拉，再降为显式试探步。
- **状态层（一级，用户无感）**：自动降级触发时，仅在降级存量 sparkline 旁显示闪烁黄点 + 状态栏文字"求解器自适应调节中"，不弹窗、不暂停仿真、不暴露 BDF/显式等档位术语。
- **一键确认（二级，罕见）**：自动降级链（BDF→后向欧拉→显式）仍失败时，自动暂停仿真并弹出**单按钮**对话框"简化模型以继续"。点击后系统**自动**执行预设降级组合（减少存量、降低耦合），用户不经手任何参数选择。
- **与既有机制边界**：
  - FR-SIM-8 处理"收敛性"压力（刚性、病态），不阻断；
  - FR-SIM-6 处理"数值域"溢出（NaN/Inf/量级突变），熔断暂停；
  - §5.1 逃生阀处理"MVP 体量超载"，由架构阶段触发，非运行时。三者层次不同，互不替代。
- **验收**：
  - 正常收敛路径：用户全程仅"播放—暂停"，零新增控件可见；
  - 一级触发：sparkline 旁黄点闪烁，仿真不暂停，用户可忽略继续；
  - 二级触发：弹单按钮"简化模型以继续"，点击后仿真从当前点续跑，无第二按钮、无参数面板。

### 3.3.1 仿真结果呈现

**FR-SIM-VIZ-1**：内联 ASCII 趋势可视化（MVP）
- 每个存量方框旁实时显示 ASCII sparkline，反映该存量近 N 步的趋势（N 可配置，默认 32）
- sparkline 字符集：`▁▂▃▄▅▆▇█`，数值自动归一化到字符高度档位
- **存量方框同步显示当前数值 + 单位（不归一化）**：与 sparkline 并存——sparkline 看趋势形状，数值 + 单位看绝对量与单位识读（兑现 §1.3 L2 渐显"单位显示"锚点）。如 `[水位]\n 1234 升`；仿真未运行时显示 `initialValue`；无 `units` 的存量仅显示数值
- **量纲识读反馈**：量纲校验结果在状态栏概要显示（见 FR-UI-3 扩展），使游戏者能"看到人口(人)随时间增长""水位(升)在下降"，而非无量纲归一化曲线
- 采样窗口与仿真步长解耦：无论 dt 如何，sparkline 固定保留最近 N 个采样点
- 纯字符渲染，与 ASCII 美学一致；不依赖任何第三方图表库
- 完整的多存量叠加时序面板（L3）为后续能力，MVP 不含

### 3.4 实时协作系统

**FR-COLLAB-1**：CRDT 文档模型
- 技术方案：Yjs CRDT 库
- 文档结构：
  - `Y.Map("elements")`：所有画布元素（id → JSON 状态）
  - 流量元素的 `formula` 以**结构化 AST** 存储（`Y.Map` 嵌套表达运算符/操作数/存量引用节点，引用节点存 stockId 见 FR-ELEM-3），而非扁平字符串——使并发编辑不同 AST 子节点可语义感知合并（FR-COLLAB-6）
  - `Y.Array("elementOrder")`：Z 轴层级排序
  - `Y.Map("boardMetadata")`：看板名称、设置、所有者
- 持久化：服务端 SQLite 存储快照 + 追加式操作日志（单节点、10 人匿名规模）

**FR-COLLAB-2**：存在感知协议
- 非持久化独立通道传输临时状态：
  - 光标位置：世界坐标 `{x, y}`
  - 选区状态：`{elementIds: [...], anchor, head}`
  - 用户信息：`{id, name, color}`
  - 心跳检测：在线状态
- 广播频率：最大 30Hz 节流
- 视口剔除：仅绘制当前可见区域内的光标

**FR-COLLAB-3**：房主迁移机制
- 指定房主（第一个用户或选举产生）运行模拟
- 若房主断开连接：
  1. 剩余用户检测到心跳丢失
  2. 先申请者当选新房主
  3. 新房主从服务端仿真态快照（FR-COLLAB-5）续跑模拟，不回退 t=0
  4. 向所有客户端广播 `SIMULATION RESUMED` 通知

**FR-COLLAB-4**：模拟状态广播
- 独立二进制通道传输模拟更新（不混入 CRDT）
- 消息格式：`[stockId<string>, currentValue<float64>]` 数组（stockId 为 UUIDv4 字符串，与 ② id 引用一致）
- 更新频率：≤ 30Hz，变化检测节流
- 广播粒度：隐式求解器（FR-SIM-1）每步经牛顿迭代收敛后的整步结果（非子迭代中间值）
- 客户端行为：接收并渲染，**不回写 CRDT**
- `currentValue` 是会话级临时状态（不进入 CRDT 文档模型、不跨会话持久化），但经独立仿真态快照通道（FR-COLLAB-5）供房主迁移恢复

**FR-COLLAB-5**：房间仿真态快照
- 房主周期性向服务端上报房间仿真态快照：`{simulationTime, stockValues, paused, dt}`
- 服务端在进程内存中持有当前房间的最新快照（不落 SQLite，会话结束丢弃）
- 房主迁移时，新房主从服务端拉取该快照续跑
- 仿真态为会话级临时态：会话内可经此机制恢复，不跨会话持久化

**FR-COLLAB-6**：公式语义感知合并
- 流量公式以结构化 AST 存于 CRDT（FR-COLLAB-1），并发编辑不同 AST 子节点时按节点级 CRDT 合并，避免文本级冲突破坏语义
- AST↔文本双向同步：公式编辑器将用户输入文本实时解析为 AST 写入 CRDT，CRDT 变化重新序列化为文本显示；不完整语法（如输入中途）以"待定节点"承载，不阻断输入
- 合并后对 AST 做语法 + 量纲校验（FR-SIM-7）；结构化合并保证 AST 始终可解析，量纲校验前置条件稳定
- 工程量警示：结构化公式 CRDT + AST↔文本同步编辑器属协作编辑器领域硬骨头，MVP 体量偏重（见 addendum §8）

### 3.5 多画板与剪贴板

**FR-BOARD-1**：标签页界面
- 每个客户端最多同时打开 5 个画板
- 超限提示 + "关闭不活跃"对话框
- 最近画板列表（最多 10 项）
- 切换画板时暂停/恢复 Y.Doc 同步

**FR-BOARD-2**：跨画板复制粘贴
- 剪贴板格式：Base64 编码 JSON + 协议头
- 协议前缀：`SD_ASCII_ENGINE://` + CRC32 校验和（CRC32 仅作完整性校验，非认证机制）
- 粘贴操作流程：
  1. 校验协议前缀合法性
  2. 校验 CRC32 校验和完整性（防传输损坏，不防伪造）
  3. **结构白名单校验**：payload 须符合图元 schema——字段类型合法、ID 为合法 UUIDv4、流量连接的 fromId/toId 引用存在于 payload 内、无未知字段；任一不合规即拒绝粘贴
  4. 为所有粘贴元素分配新 UUID
  5. 重映射流量连接到新 ID
  6. 位置偏移 (x + 2, y + 2) 单元格
  7. 插入空间索引
- 原子事务：整个粘贴作为单个 CRDT 操作
- 安全边界：粘贴通道仅交付图元结构，不经 eval；防护目标为"拒绝畸形/非法图元结构"，非密码学认证

**FR-BOARD-3**：外部文件阻断
- 拖放文件上传：阻止所有类型文件的默认行为
- 外部粘贴：仅接受带有有效协议前缀的纯文本
- 图片/二进制数据：静默拒绝

- 国际化：界面文本中/英双语可切换（FR-UI-4）；MVP 仅此两种语言，i18n 框架需为后续扩展语言预留

## 五、MVP 范围与路线图

### 5.1 MVP 包含（第一阶段）

✅ ASCII 渲染无限画布  
✅ 存量、源/汇、流量 三种图元（SFD）  
✅ 隐式求解器（后向欧拉/BDF + 牛顿迭代）+ 存量断路器算法 + 求解器自适应降级（FR-SIM-8）
✅ 量纲一致性校验（units + 公式量纲推导 + 软警告）+ 模型级时间单位  
✅ CRDT 实时多人协作（含公式语义感知合并）  
✅ 房主计算 + 迁移机制（从服务端仿真态快照续跑）  
✅ 跨画板复制粘贴（带协议校验）  
✅ 画板独立撤销栈  
✅ 存在感知（光标 + 选区跟随）  
✅ 内联 ASCII 趋势可视化（sparkline）  
✅ 可选游戏化层：行为徽章系统 + 总开关  

> **[体量逃生阀]** 上述 MVP 含四项重决策（隐式求解器 + 量纲校验 + 公式语义合并 + 游戏化），叠加后实质偏重（警示记录见 addendum §7.3 / §8.3）。若架构阶段核算判定 MVP 范围超载，按以下优先级降级，降级后仍维持核心产品故事（ASCII SFD 建模 + 多人协作 + 数值模拟）：
> 1. **⑤ 公式语义合并** → 降为冲突检测 + 提示（方案 A）
> 2. **③ 隐式求解器** → 降为自适应 RK45
> 3. **① 量纲校验** → 降为显示用 units（方案 B）
> 4. 跨画板复制粘贴 → 移至后续版本
>
> 降级优先级 ⑤ > ③ > ①（对模型洞察力损伤由小到大）。详见 addendum §8.3。

### 5.2 MVP 排除（后续版本）

❌ 重型第三方图表库（ECharts/D3）集成——MVP 以纯 ASCII 趋势呈现仿真结果；完整多存量时序面板（L3）为后续  
❌ 因果回路图（CLD）定性因果图——MVP 仅 SFD  
❌ 挑战关卡——作为紧随 MVP 之后的第一优先项，依赖仿真结果数值比对引擎  
❌ 离线编辑模式  
❌ 模板库市场  
❌ 导出 PNG/SVG/PDF  
❌ 用户账户与权限系统（MVP 匿名模式）  
❌ 移动端响应式布局  
❌ 模拟历史回放  
❌ 参数滑块与敏感性分析

## 六、开放问题与假设

**[假设 1]** 模拟主机为第一个进入房间的用户，断开时自动迁移。替代方案：服务器端统一计算。

**[假设 2]** 粘贴后自动选中行为推迟到实现阶段。建议：不自动选中，减少 Awareness 网络流量。

**[假设 3]** 粘贴原子性与依赖操作的处理策略待定。选项 A：硬阻断，粘贴完成前禁止连线；选项 B：允许连线但撤销时级联删除。

**[决策（原开放问题 1）]** 仿真态为会话级临时态：会话内可经服务端仿真态快照（FR-COLLAB-5）恢复以支持房主迁移；不跨会话持久化。会话结束即丢弃运行时状态，下次打开画板从模型定义的初始值重新开始。

> *原开放问题 1：仿真运行时态是否跨会话持久化？*

**[开放问题 2]** 官方支持的最大画板尺寸？当前目标：10,000 图元。

**[开放问题 3]** MVP 仅匿名协作，还是需要用户认证系统？（影响 FR-GAME-2 徽章记录与画板归属的持久化）

**[开放问题 4]** 挑战关卡的达标判定引擎与 L3 时序面板是否复用同一仿真结果数据通道？二者均为后续能力，需在架构阶段统一数据通路设计（详见 addendum）。

## 八、术语表 (Glossary)

> 汇总本 PRD 使用的域名词与七大数学约束，供架构 / 故事阶段抽取单个 FR 时快速查阅。约束机制详见 §2.2 表，选型论证见 `addendum.md`。

### 域名词

| 术语 | 全称 / 释义 | 定义出处 |
|------|------------|----------|
| **SFD** | Stock-Flow Diagram（存量-流量图）：本工具 MVP 唯一建模图类型 | §5.1 |
| **CLD** | Causal Loop Diagram（因果回路图）：定性因果图，MVP 排除，留后续 | §5.2 |
| **CRDT** | Conflict-free Replicated Data Type（无冲突复制数据类型）；本工具用 Yjs `Y.Map`/`Y.Array` 实现协作 | FR-COLLAB-1 |
| **R 树** | R-tree 空间索引，配合视口剔除加速 10k 图元画布命中测试 | §七 渲染层 |
| **VRAM 双缓冲** | 文本缓冲 + 颜色缓冲双区，逐帧合成 ASCII 渲染 | §七 渲染层 |
| **sparkline** | 内联 ASCII 趋势字符（▁▂▃▄▅▆▇█），与存量数值+单位并存的趋势呈现 | FR-SIM-VIZ-1 |
| **Awareness** | Yjs 存在感知通道，承载光标 / 选区跟随 | FR-COLLAB-3 |

### 七大数学约束（见 §2.2）

| 约束 | 一句话定义 |
|------|-----------|
| **存量断路器算法** | 拓扑排序前强行剪断所有存量流出边，区分积分反馈环与代数死锁环 |
| **单次求值原则** | 所有流量速率每步只算一次，查表供所有存量使用，消除浮点误差累计 |
| **非负钳制机制** | 存量 < 0 时限制流出并级联影响下游（隐式法下经活动集 / 投影实现） |
| **DELAY 隐式存量** | 延迟函数不做历史队列，转为串联幽灵存量，保证隐式联立求解连续性 |
| **双时间轴分离** | UI 时间轴 ≠ 仿真时间轴，仿真数值异常不传导到画布渲染 |
| **NaN 熔断保护** | 每步收敛后做 NaN/Inf 位运算检查 + 相对量级突变检查（FR-SIM-6） |
| **两遍扫描初始化** | Pass0 算初始值 → Pass1 焊死后推导流速，解决 t=0 时序死锁 |

### 求解器运行时机制（非 §2.2 约束，运行时行为）

| 机制 | 一句话定义 |
|------|-----------|
| **求解器自适应降级 + 一键简化**（FR-SIM-8） | BDF 收敛监控→自动降级后向欧拉/显式试探步（黄点指示，无感）→仍失败弹单按钮"简化模型以继续"，用户零档位介入 |

> 命名一致性：`NaN 熔断保护` / `NaN 熔断` / `数值溢出熔断`（FR-SIM-6）指同一机制，统一以 §2.2 `NaN 熔断保护` 为准。

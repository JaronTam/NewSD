# Project Context for AI Agents

_This file contains critical rules and patterns that AI agents must follow when implementing code in this project. Focus on unobvious details that agents might otherwise miss._

## Technology Stack & Versions

### Stack map (load-bearing deps -> AD)

- React 19 + TanStack Start 1.x (SPA) + TanStack Router - render+route, AD-1
- TypeScript 5.8 strict - `src/`
- Rust + wasm-pack (crate `newsd-core` @ `wasm/`, currently a stub: `wasm-bindgen` only; `faer`/`autodiff` are epic-1b targets) - AD-5/AD-6
- Go single binary (repo-root `main.go` + `internal/{auth,server,store,version}/`; 1a.1 skeleton + 2.1 auth built; NOT `server/` subdir, SDR#7 不纠正) - AD-3/AD-18
- SQLite WAL in-process - AD-2

Exact patch versions: `package.json` / `wasm/Cargo.toml` / (future) `go.mod` are authoritative-don't duplicate here.

### Version & supplier constraints (non-obvious)

1. **AD-bound, not arbitrary.** Stack versions are architectural, governed by the 18 ADs. `vite.config.ts` keeps a committed stack-table literal ("TanStack Start ^1.168.26"). Never bump a load-bearing dep without an AD update + architecture review.
2. **Load-bearing vs utility.** Only AD-mapped deps above are AD-bound; utilities (rbush, clsx, lucide, cva, tailwind-merge) follow normal semver-don't block a util bump on an AD review.
3. **Supplier-fragile (forward-gated).** `faer` 0.24.4 / `autodiff` 0.7.0 / `yjs-go` are targets, not yet declared (`wasm/Cargo.toml` currently has only `wasm-bindgen`; Go backend built at repo root, yjs-go still epic 2+ target not yet declared). Gate their introduction on epic 1b/2+; on upgrade re-run the numeric correctness suite (exists only after 1b). `yjs-go` stalls -> self-built Go binding, not Node (AD-4). `meval` forbidden (AD-6).
4. **Runtime = bun.** `bun.lock` is the committed lockfile; `package-lock.json` is gitignored ("npm 残留", forbidden)-delete it if it appears. README documents bun; npm/pnpm scripts execute but must not write a competing lockfile or use bun-only source APIs that break npm.
5. **Pre-2.0 pin risk.** TanStack Start (1.x) and `faer`/`autodiff` (0.x) are pre-stable; `^` allows breaking minors. Prefer exact/tilde pin for these; let the lockfile carry exact resolution.
6. **TanStack SPA coupling.** `server-only` is eslint-forbidden (use `*.server.ts` or `@tanstack/react-start/server-only`); server functions exist in the API but SPA has no server runtime-don't author logic depending on server-side execution. vitest config is intentionally separate from vite.config (the TanStack plugin must not run at test time-generates `routeTree.gen.ts`, expects a router entry). `routeTree.gen.ts` is eslint-ignored, regenerated by build-don't hand-edit.
7. **Wasm boundary (AD-5).** All sim-step numeric eval -> Rust kernel (`wasm/`); TS is a thin boundary-never implement integration/Newton/autodiff/LU in TS. Wasm load-gate: on load failure surface `[SYSTEM HALTED]` (AD-5 circuit breaker), not a blank screen. In vitest (jsdom, no WASM) mock the Wasm seam-don't skip sim tests.
8. **Preview host pinned `127.0.0.1`** (vite.config.ts:14-21): prerender crawls a random-port preview server; default `localhost` can split IPv4/IPv6 inside buildx -> ConnectionRefused. Don't "clean up" to `localhost`.

### Forward-looking (epic 1b/2+ - verify when reached)

- **Build coupling (AD-3).** Future `go build` embeds `dist/` via `go:embed` -> build frontend first; fresh-clone order = install + `bun run build` before `go build`. CI must serialize or artifact-pass `dist/`.
- **SQLite driver = `modernc.org/sqlite` v1.53.0** (1a.1 引, pure-Go no-cgo 保 AD-3 单 binary; 2.1 users/sessions 表 via `internal/store/schema.go` Migrate). `mattn/go-sqlite3` (cgo) rejected.
- **Security advisory overrides the lockfile rule.** A transitive advisory justifies deliberate regeneration with `bun audit` review (not the routine "don't regenerate").

## Critical Implementation Rules

### Language-Specific Rules

**TypeScript (`src/`)**

- `strict: true`, target ES2022, `moduleResolution: Bundler`. No `any` without justification; prefer discriminated unions over casts.
- Path alias `@/*` -> `./src/*` is resolved natively by Vite (`resolve.tsconfigPaths: true`)-do NOT add a `tsconfig-paths` plugin or manual alias. `allowImportingTsExtensions` is on-explicit `.ts`/`.tsx` import suffixes are allowed and used.
- `verbatimModuleSyntax=false`: `import type` is optional, not enforced.
- **Unused tolerance**: `noUnusedLocals=false` + `noUnusedParameters=false` + eslint `@typescript-eslint/no-unused-vars: off`. Unused identifiers do NOT fail the build-do not "clean up" flagged-unused in a PR unless the code is actually dead.
- **Discriminant field is context-specific, not blanket**: `SDElement` narrows on `el.kind` (`"stock"|"cloud"|"flow"`); toast messages and some other unions use `type`. Match the existing discriminant of the union you extend-don't mix within one union. (Epic/spec prose says `type`; SDElement code uses `kind`-narrow on `kind`.)
- **Element identity = `id` (UUIDv4 via `crypto.randomUUID`)**. Architecture docs call this `nid`; the code field is `id`. Per-type `seq` is display-only (SDR#1-14); flow endpoints reference `fromId`/`toId` (id-keyed). Never key Maps/refs/CRDT by `seq` or `name`-only by `id`.
- **Error handling = `throw new Error(message)`** for invariants, invalid args, parse failures, and WebGL/state null-checks. No custom error subclasses or error codes. Tests narrow with `throw new Error("unreachable: ...")`. Don't swallow; don't introduce error-class hierarchies.
- **Exhaustiveness**: when `switch`-ing over `ElementKind`/`kind`, assert exhaustiveness via a `default: const _exhaustive: never = el; return _exhaustive;` guard so adding a new kind fails at compile time. ⚠ This pattern is NOT yet adopted repo-wide-existing switches (`store.ts:100`, `store.ts:144`, `minimap.ts:247`) lack it, so adding a new `kind` today is a SILENT SKIP, not a compile error. Adopt the `never` default when touching those switches.
- **`crypto.randomUUID` requires a secure context** (https or localhost). Dev is localhost (ok); non-localhost http deploys break id generation-and OAuth (AD-16) too. Don't replace with a non-secure fallback without a decision.

**Rust (`wasm/`)**

- Edition 2021, `crate-type = ["cdylib", "rlib"]`, `wasm-bindgen` 0.2 is the only TS<->Rust boundary.
- Types across the boundary are hand-synced (no codegen yet)-changing an exported `#[wasm_bindgen]` signature requires updating the TS binding manually.

**Go (repo root `main.go` + `internal/`; 1a.1 skeleton + 2.1 auth)**

- Implemented: 1a.1 store/server/version skeleton + 2.1 OAuth auth (`internal/auth/` users/sessions tables, session dual-channel). NOT `server/` subdir (1a.1 deviated from spine; SDR#7 不纠正). See Backend auth section below. Go language rules grow as backend expands; revisit per story.

### Framework-Specific Rules

**React 19 + state**

- **State = `useSyncExternalStore` + external singleton stores** (`elementStore`, `promptStore`), NOT Redux/Zustand/Jotai/react-query. A store exposes `subscribe` + `getSnapshot`; components read via `useSyncExternalStore(store.subscribe, store.getSnapshot)`. Extending the store adapter is the pattern-don't introduce another state lib or query cache.
- **Snapshot immutability is load-bearing**: `getSnapshot` returns the store's cached `elements` ref (stable between mutations). Every mutator MUST reassign `elements = [...]` (new array)-never `push`/in-place mutate-or React skips the update (no re-render) and tearing can occur. (Verified: `createStock`/`deleteElement`/`setElements` all reassign.)
- **`subscribe` is Set-based** (idempotent add/delete). StrictMode is currently OFF, but the Set design is StrictMode-safe if enabled-don't switch to an array-based listener list.
- React 19; no experimental-feature opt-in (`use`, Actions, etc.) without a decision.

**State persistence (1a.11 seq + 1a.13 autosave)**

- **1a.11 seq storage = in-memory counter + `setElements` `deriveSeq` load-承接** (NOT persisted): `deriveSeq` 载入时承接 existing seq (双端锚定正则 + `MAX_SAFE_INTEGER` 边界); rename 脱规范 seq 永久流失 (acceptable). Per-type `seq` is display-only (see Language Rules §element-identity=id is the key).
- **1a.13 autosave = localStorage** (F2 `beforeunload` flush + F3 定时 autosave; A2 hydrate 天然兼容-载入 localStorage elements). F5 刷新保护 = beforeunload flush + hydrate restore. Impl: `src/lib/sd/autosave.ts`.
- **ATDD red-scaffold for new modules** (1a.13): 全新模块 import fail 破坏 baseline tsc 0; `declare const` ambient 声明 + `it.skip` 兼容 (1a.13 `autosave.ts` 首例: 23 `it.skip` + 2 e2e skip, tsc 0, baseline passed count 不变). DS 首步换真实 import.

**Canvas rendering (AD-9)**

- `CanvasView` renders via WebGL2 (`src/lib/render/vram/renderer.ts`); entities are GPU-drawn, NOT DOM elements. There is **no DOM overlay** on the canvas-don't add one "for testability" (breaks the render model; 1a.8 e2e lesson: DOM selectors can't reach canvas content).
- e2e/queries against canvas content must use canvas-coordinate probing or app state, not DOM selectors.
- **WebGL context loss is NOT handled** (no `webglcontextlost` listener). On context loss the render silently freezes. Forward gap: handle `webglcontextlost` -> surface `[SYSTEM HALTED]` (AD-5 circuit breaker), mirroring the WebGL2-unavailable throw path (`renderer.ts:200`).

**Tailwind v4**

- CSS-based config: there is NO `tailwind.config.{js,ts}`. Theming lives in `src/styles/tokens.css` (`@theme` / `--ns-*` tokens) consumed via `@tailwindcss/vite`. Don't add a JS config file.
- className composition uses `cn()` (clsx + `tailwind-merge`) from `@/lib/utils`. `class-variance-authority` is installed but NOT used in `src/`-don't assume CVA is the variant convention; use `cn()`.

**TanStack Router**

- File-based routing in `src/routes/` (`__root.tsx`, `index.tsx`, `vram.tsx`). `routeTree.gen.ts` is build-generated (eslint-ignored)-add routes as files, don't hand-edit the tree.
- SPA mode: server functions are inert (see Technology Stack §6)-don't author server-side execution.

**Backend auth (2.1 OAuth, Go + frontend)**

- **client_secret SERVER-ONLY (SDR#6 硬红线)**: NEVER 入前端 `dist/`/`src/`; 仅 Go backend env. 泄漏=不可逆(须 provider 轮换).
- **手搓 OAuth 零新依赖 (SDR#1)**: 无 `golang.org/x/oauth2`; stdlib `net/http`+`encoding/json`+`crypto/rand`+`crypto/subtle`+`encoding/base64`. web flow = authorize URL -> code -> token endpoint -> userinfo (GitHub `login/oauth/access_token`+`api.github.com/user`; Google `oauth2.googleapis.com/token`+`www.googleapis.com/oauth2/v2/userinfo`).
- **Session 双通道**: cookie (session id, HttpOnly + SameSite=Lax AD-16, Secure 条件化) + DB `sessions` 行 (AD-16 ERD). IR §476 backup/restore fold-in. `users`/`sessions` 表 via `internal/store/schema.go` Migrate.
- **Secure cookie 条件化 (Q3=A)**: dev (localhost) 跳 Secure (浏览器+OAuth localhost http 豁免); prod 设. SameSite=Lax 是 CSRF 主防线 (AD-16); 自签 cert 拒 (过复杂).
- **FAKE_OAUTH=1 dev-only fake provider (Q2=B)**: server 仅 `FAKE_OAUTH=1` env 时注册 dev-only fake route; 生产未设不注册 (grep 生产 binary 无暴露). e2e 全流程 (login -> fake authorize -> callback -> session cookie -> `/api/auth/me`). 模式对镜 1a.5 `window.__e2e__` 注入但 server 侧.
- **canvas login gate defer (Q1=A, SDR#35/AC-16)**: 2.1 不 gate canvas; canvas 强制 gate defer 2.2 (board 归属需 login). AC-16 维持现状.
- **login route** = `src/routes/login.tsx` (非 index `/` = CanvasView); `/api/auth/me` 返 session user.

### Testing Rules

**Three-tier pyramid (don't blur the tiers)**

- **Unit (vitest, jsdom)**: `src/**/*.test.{ts,tsx}` (31 files). Pure domain logic + pure render helpers (camera math, palette, shader math). Run: `bun run test`. Config: `vitest.config.ts`-intentionally SEPARATE from `vite.config.ts` (the TanStack Start vite plugin must NOT run during tests; it generates `routeTree.gen.ts` and expects a router entry).
- **Component (vitest, jsdom + @testing-library/react)**: `getContext` is stubbed to `null` in jsdom, so every canvas-drawing component MUST null-guard `getContext` and early-return `draw()` on null-assert observable side-effects (HUD text, store state), NOT canvas pixels.
- **E2E (Playwright, chromium)**: `e2e/*.spec.ts` (8 files). Real WebGL2 via SwiftShader. Run: `bun run test:e2e`.

**jsdom has NO WebGL2 (load-bearing test split)**

- Don't mock WebGL2 in jsdom. `renderer.ts` exports pure-logic helpers (`affineToMat3`, `paletteToUniform`, `quadWorldSize`, ...) so the math is unit-tested; the GPU draw path is verified ONLY via Playwright e2e. Factor new pure math out-don't instantiate `VRAMRenderer` under jsdom.
- The constructor's null-ctx throw (`renderer.ts:200`) IS unit-tested (refuses a null context).
- When real Wasm lands (epic 1b), add a seam mock in `src/test/setup.ts`-vitest cannot import `.wasm` directly.

**Playwright e2e non-obvious**

- WebGL2 in headless/CI needs SwiftShader: `launchOptions.args = ["--use-gl=angle", "--use-angle=swiftshader"]` (`playwright.config.ts:24`). Load-bearing-don't drop "to simplify" or WebGL2 e2e fails.
- `preserveDrawingBuffer: true` (`renderer.ts:198`) exists for Playwright screenshot capture-don't "optimize" it away or screenshot e2e goes black.
- `webServer.command: "npx vite"` (port 5173)-e2e boots its own dev server.
- Visual regression (AC-16): `snapshotDir: ./e2e/__snapshots__`; update via `npx playwright test --update-snapshots` (no script in `package.json`).

**jsdom stubs (`src/test/setup.ts`)** - if you add a browser API jsdom lacks, stub it HERE, not in the test file:

- `matchMedia` (`matches:false`), `ResizeObserver` (fires callback once, zero-size), `HTMLCanvasElement.prototype.getContext` -> `null`.

**Pre-merge gate is LOCAL (no CI workflow exists)**

- There is NO `.github/workflows/`; the `process.env.CI` branches in `playwright.config.ts` are aspirational/dead. Nothing runs these for you-the gate is a LOCAL manual run, all green:
  - `bun run typecheck` (`tsc --noEmit`) - 0 errors.
  - `bun run test` (vitest) - 0 failures.
  - `bun run test:e2e` (Playwright) - 0 failures.
  - `bun run lint` (`eslint .`) - 0 errors.
- `src/routeTree.gen.ts` is GENERATED but COMMITTED (git-tracked, not gitignored; `tsconfig` includes `src/**/*.ts`). Don't gitignore it or fresh-clone `bun run typecheck` breaks.

**Record FULL counts, passed AND skipped (not subsets)**

- In sprint-status/CR, record the full suite count including skips (e.g., `730 passed / 1 skipped` vitest, `29 passed / 21 skipped` e2e), NOT the new story's subset (`7/7`). AC-8 "no regression" = full suite green.
- A skip is NOT a pass: every `.skip` needs a reason comment + `deferred-work.md` entry (1a.12: 21 skipped e2e = D4 canvas-click infra deferred). `forbidOnly: !!CI` rejects `.only`-don't commit it.

**Story-cycle testing discipline (BMAD)**

- DS step8 self-attestation ("AC/SDR PASS") MUST cite test evidence; CR Run 1 may still FAIL-self-eval is not the CR verdict.
- ATDD: run `/bmad-testarch-atdd` to scaffold red tests BEFORE DS (1a.5+). Verifies the render-architecture assumption (DOM overlay vs WebGL canvas)-1a.8 lesson: DOM-selector e2e against a WebGL canvas is not achievable, defer or re-scope.

**Wasm tests**

- `wasm-pack test` (crate has `wasm-bindgen-test` dev-dep; wasm-pack 0.15 local). Stub crate now (see Technology Stack)-grows with epic 1b.

### Code Quality & Style Rules

**Formatting (prettier, enforced via eslint)**

- `.prettierrc`: `printWidth: 100`, `semi: true`, `singleQuote: false` (**DOUBLE quotes**-non-obvious, many default to single), `trailingComma: "all"`. Run `bun run format` (prettier --write); don't hand-format. Prettier runs AS an eslint rule (`eslint-plugin-prettier/recommended`), so `bun run lint` also fails on format drift-run format then lint. No competing formatter (biome/deno fmt).
- `.prettierignore` excludes dist/.output/.tanstack/coverage/wasm-target/wasm-pkg/routeTree.gen.ts/lockfiles.

**Styling = hand-written `.ns-*` BEM classes (NOT inline Tailwind utilities)**

- Components style via semantic `.ns-*` BEM classes defined in `src/styles.css` (block, `block__element`, `block--modifier`, `block__element--modifier`-e.g., `ns-toolbar__btn--active`, `ns-prompt-panel__handle`). `className` references these names. **Don't sprinkle inline Tailwind utility classes** "for a quick tweak"-add a `.ns-*` class to `styles.css` and match the BEM structure.
- `cn()` (`@/lib/utils`, clsx + `tailwind-merge`) is the className COMPOSER for conditional toggles (e.g., `${cond ? " ns-toolbar__btn--active" : ""}`); the class VALUES are `.ns-*` names, not utilities. CVA is installed but unused (see Framework Rules).
- Tailwind v4 `@theme inline` in `src/styles.css` maps `--ns-*` tokens (`src/styles/tokens.css`) to `--color-*` utilities-the `.ns-*` CSS consumes `var(--ns-*)`. Theming = edit tokens.css, not component classes.

**eslint (flat config, `eslint.config.js`)**

- Extends `@eslint/js` + `typescript-eslint` recommended; plugins `react-hooks` (recommended) + `react-refresh`.
- `@typescript-eslint/no-unused-vars: off` - unused does NOT fail lint (see Language Rules §unused-tolerance); don't "clean up" flagged-unused unless code is dead.
- `no-restricted-imports` bans `server-only` (use `@tanstack/react-start/server-only` or `*.server.ts`).
- `react-refresh/only-export-components: warn` (allowConstantExport). `ignores`: dist, .output, .tanstack, coverage, wasm/target, wasm/pkg, routeTree.gen.ts.

**TypeScript strictness extras (tsconfig.json; strict/error-handling see Language Rules)**

- `noFallthroughCasesInSwitch: true` - every `case` must `break`/`return`/`throw`; accidental fallthrough is a compile error.
- `noUncheckedSideEffectImports: true` (TS 5.6) - an unresolvable side-effect `import "x"` is a compile error (catches CSS/asset import typos).
- `allowImportingTsExtensions: true` - explicit `.ts`/`.tsx` import suffixes allowed and used. `target: ES2022`, `moduleResolution: Bundler`, `noEmit: true`.

**Naming conventions (convention, NOT lint-enforced)**

- Files: PascalCase for components (`CanvasView.tsx`, `PromptPanel.tsx`); camelCase for logic modules (`store.ts`, `errorDetection.ts`). Case-sensitive on disk.
- Identifiers: camelCase functions/vars (`createStock`, `deriveSeq`); PascalCase types/classes (`SDElement`, `VRAMRenderer`); UPPER_SNAKE constants (`PALETTE_SIZE`, `INITIAL_CAPACITY`, `VERTS_PER_INSTANCE`).
- CSS tokens: `--ns-*` prefix (kebab) in `src/styles/tokens.css`.
- No eslint naming rule exists-match the existing convention of the directory; don't add one without a decision.

**Comments & spec-traceability (load-bearing, convention-not-enforced - HIGH RISK)**

- Public functions get a `/** ... */` JSDoc line; inline notes use `//`.
- Comments CITE spec anchors pervasively: `// AD-9 ...`, `/** SDR#9: ... */`, `/** AC-16/AC-12: ... */`, `// Story 1a.2 sub-PR #1`, `/* per CS钉死 #2 */`. This maps code to spec/epic decisions. ⚠ No lint rule enforces this-an agent stripping anchor comments "as cleanup" silently destroys traceability. Preserve: if a function implements an SDR/AC, its comment should name it. NEVER strip anchor comments.
- `switch` over `kind`: assert exhaustiveness via `default: const _exhaustive: never = el;` (see Language Rules). ⚠ Not yet adopted repo-wide-see Language Rules for the gap note (existing switches silently skip new kinds).

### Development Workflow Rules

**Story-cycle (BMAD, formalized CS->VS->DS->CR)**

- Authority: `epics.md` (story-cycle section) + `_bmad-output/planning-artifacts/story-cycle-formalization.md`. Stories 1a.3+ run the 4-phase cycle. NOT applicable to: spikes (local branch, not merged), prototype probes (pre-epic), doc-only chore PRs (sprint-status, glossary) - those go direct.
- Invoke skills BY MECHANISM, not hand-written prompts. Truth source skill↔action: `_bmad/bmm/module-help.csv`. Inspect a skill: `.claude/skills/<name>/**`.
  - CS=`bmad-create-story` / VS=`bmad-create-story validate` (= `*validate-create-story`, a validate action not a separate dir) / DS=`bmad-dev-story` / CR=`bmad-code-review`.
- VS must leave an EXPLICIT record in the story, even if zero findings ("VS PASS, no findings"). 1a.5 left no persisted artifact; 1a.6+ required. VS gate also checks: CS web-research explicit (new deps: version+why+breaking; no new deps: no-op citing baseline version lock) AND task↔SDR consistency (1a.7: DS followed task line, not SDR#7).

**DS discipline**

- DS follows the SDR, not the task line. 1a.7: task T11 said `createStock`, SDR#7 said "new=full clear"; DS implemented createStock, contradicting SDR. VS gates task↔SDR consistency (see VS rules above), but DS still self-checks-don't rely solely on VS catching it.
- DS step8 self-attestation ("AC/SDR PASS") MUST cite test evidence (file:line / test name). Self-eval ≠ CR verdict-CR Run 1 may still FAIL (1a.5).
- DS step8 baseline diff review: `git diff <baseline_commit>..HEAD` 逐文件核,落 story `### step8 baseline diff review` 逐文件核验表 (`文件 | Dev Log 声明 | diff 实际 | 一致?`);声明↔diff 不一致当场修,禁过 step8 (1a.8 F-1 教训: `deriveFlowUnits` 声明 vs 实读 `selectedElement.units` 无此表存活到 CR)。详 `story-cycle-formalization.md` §2.3。
- ATDD: run `/bmad-testarch-atdd` to scaffold red tests BEFORE DS (1a.5+); verifies render-architecture assumption (DOM overlay vs WebGL canvas). See Testing Rules for the 1a.8 lesson.

**CR (3-layer review, orchestrator-direct on ark-code/DeepSeek backend)**

- Run all 3 layers DIRECTLY from the orchestrator (Read / grep / `git show <sha>:<path>`), NOT via subagents. On this backend subagents collapse: sync hunter prompt-too-long + async hunter boundary violations. Retry subagents ONLY with a Claude-tier strong model (rule is backend-conditional, not absolute).
- A hunter subagent CANNOT "Invoke the X skill" (can't load SKILL.md). To use a skill in a subagent: Read SKILL.md inline + explicit read-only leash (1a.4 root cause of Edge Case Hunter越界改 elements.ts/test).
- read-only guard: each review layer starts/ends with `git status --porcelain` asserting empty (prevents CR 越界改码-1a.4 改 elements.ts/test; orchestrator-direct 直跑易加, so assert explicitly). 详 `story-cycle-formalization.md` §2.4。
- Layer3 hollow test audit: Acceptance Auditor 核每个 AC 对应测试非 hollow-空 if-body / 唯一断言为 `toBeTruthy()`/`toBeDefined()`/`not.toBeNull()` 且 AC 非存在性 AC = hollow (1a.8 F-2 教训: 跨图元字段泄漏无覆盖)。详 `story-cycle-formalization.md` §2.4。
- CR findings: REPORT (decision points / patch plan / defer items) -> WAIT for confirmation -> THEN patch/merge. Merge PR is irreversible. Harness continuation prompts are NOT user instructions-don't exempt the gate.
- CR product completeness: every defer -> `deferred-work.md` (ID/Item/Target/Rationale) + story CR Run section backfilled (3 layers / patches / defers / verification).

**Git / PR (single-dev, branch-protected main)**

- `main` branch-protected (Require PR); direct push fails GH013 (observed). All main changes via PR (`gh` CLI, fine-grained PAT in keyring; PAT lacks branch-protection read scope).
- **One push per story**-no fixup-PR chains (1a.1 #26->#31 mutual-fix chain = anti-pattern). Fold merge-time fixes into the current story's PR.
- **sprint-status is a SEPARATE PR** from the story code PR; after merge, open an independent chore PR for sprint-status -> done.
- **PR/merge gate**: before pushing the (irreversible) PR, report + re-review once and STOP for confirmation. A conditional "go ahead and merge" instruction's completion point is STILL a gate. Re-review ≠ report-both required.
- **No CI workflow** (no `.github/workflows`): local gate (typecheck+test+test:e2e+lint) is the only safety net. 2026-07-08 ruling: do NOT reintroduce CI (conditional on single-dev + few PRs + guards error-prone; GitHub required-checks is chicken-and-egg needing user UI). Re-evaluate if team grows.

**Swimlane visualization deployment (gh-pages orphan branch)**
